## 1. MemPalace: LLM 记忆的“记忆宫殿”空间隐喻
- 来源：arXiv cs.AI (2604.21284)
- 链接：https://arxiv.org/abs/2604.21284
- 一句话解释：利用古老的“记忆宫殿”法（Method of Loci），将 LLM 的长短期记忆映射到虚拟空间坐标中，实现极高性能的检索。
- 它为什么重要：它把纯数学的向量检索（Vector Search）变成了**空间几何问题**。对于学艺术和设计的人来说，这让“看不见的数据”变成了“看得见的建筑”。
- 技术趋势标签：#HCI #SpatialComputing #LLM-Memory
- 可复用模式：**数据空间化（Data Spatialization）**。将非结构化信息映射到物理/几何坐标系，利用空间邻近性代替语义相似度。
- 疯狂发明家灵感：
  1. 一个小实验：用 Minecraft 的坐标系存储你的笔记，走近某个建筑时，LLM 自动加载关联的知识。
  2. 一个中型项目：一个 3D 知识库，用户在 VR 中行走，通过物体的空间排列来“编程”Agent 的长期记忆。
  3. 一个艺术/游戏/叙事方向：一个会根据玩家在游戏世界中移动轨迹来“遗忘”和“记起”剧情的 NPC。
- 对我项目的启发：你的 Minecraft/世界生成项目可以不只是背景，它可以是 Agent 的**物理脑回路**。
- 今天是否值得深入研究：**是**
- 建议行动：做demo（尝试将 RAG 的 Embedding 映射到 3D 散点图并可视化行走路径）。

## 2. Co-Evolving Skill Bank: 像生物一样进化的 Agent 技能库
- 来源：arXiv cs.AI (2604.20987)
- 链接：https://arxiv.org/abs/2604.20987
- 一句话解释：Agent 在长程任务（如游戏）中，不仅在做决策，还在不断自我生成、迭代和重写自己的“技能代码库”。
- 它为什么重要：它打破了“写死代码”的逻辑，Agent 拥有了**“肌肉记忆”的数字版本**。
- 技术趋势标签：#Agent #GameSystems #Self-Evolving
- 可复用模式：**技能解耦与重组**。不要给 Agent 一个大 Prompt，而是给它一个可以不断长大的 `functions/` 文件夹。
- 疯狂发明家灵感：
  1. 一个小实验：写一个 Agent，它每完成一次任务就总结一段 Python 函数存入本地，下次任务优先调用。
  2. 一个中型项目：一个 Minecraft 机器人，它从“乱跳”开始，通过进化逐渐学会“建造哥特式尖顶”的复杂技能链。
  3. 一个艺术/游戏/叙事方向：一个会“学习”玩家操作风格并逐渐进化出对抗性技能的数字生命模型。
- 对我项目的启发：强化你的 spec-driven 理念——不仅生成内容，还要生成**生成内容的能力**。
- 今天是否值得深入研究：**是**
- 建议行动：试用（在你的 Agent 工作流中加入一个 `skill_manager` 模块）。

## 3. WorkflowGen: 轨迹驱动的自动化流生成
- 来源：arXiv cs.LG (2604.19756)
- 链接：https://arxiv.org/abs/2604.19756
- 一句话解释：不再从头写 Workflow，而是通过观察人类或 Agent 乱撞的“执行轨迹”，自动提取出最稳健的自动化流程图。
- 它为什么重要：解决了 Spec-driven development 中“Spec 难写”的问题。**乱撞的过程就是 Spec 生成的过程。**
- 技术趋势标签：#Automation #Spec-Driven #Workflow
- 可复用模式：**轨迹提纯（Trajectory Distillation）**。从混乱的交互记录中提取逻辑骨干。
- 疯狂发明家灵感：
  1. 一个小实验：录制你操作浏览器的 10 分钟过程，让 LLM 把它转化成一个标准的 JSON 自动化工作流。
  2. 一个中型项目：一个“影子编程”工具，它观察你写 Creative Coding 的逻辑，自动生成一套可复用的参数扰动框架。
  3. 一个艺术/游戏/叙事方向：一个互动叙事系统，它根据前 100 名玩家的通关轨迹，自动生成第 101 名玩家的“最优/最难剧情路径”。
- 对我项目的启发：关注 Drift 和自动化系统时，考虑如何把“失败的尝试”也作为生成 Spec 的养料。
- 今天是否值得深入研究：**否（读摘要即可）**
- 建议行动：收藏。

## 4. Bongard-LOGO: 符号接地与抽象视觉推理
- 来源：arXiv cs.AI (2604.21346)
- 链接：https://arxiv.org/abs/2604.21346
- 一句话解释：通过比较 VLM（多模态模型）在处理“生成程序”产生的图形时的表现，探讨 AI 是否真的理解几何逻辑。
- 它为什么重要：它将 **LOGO 语言（经典的绘图语言）** 与现代 VLM 结合。这对于 Creative Coding 极其重要：AI 应该直接画图，还是写代码画图？
- 技术趋势标签：#CreativeCoding #GenerativeSystems #Reasoning
- 可复用模式：**代码作为视觉中介（Code as Visual Intermediate）**。用程序语言来定义视觉规则，而不是靠像素。
- 疯狂发明家灵感：
  1. 一个小实验：让 LLM 写一段 p5.js 代码来描述“忧郁的圆形”，看它如何通过参数来表达抽象情感。
  2. 一个中型项目：一个基于 LLM 的“生成式画笔”，你描述意图，它实时修改 underlying 的几何生成算法。
  3. 一个艺术/游戏/叙事方向：一个完全由几何逻辑驱动的世界，所有地形和生物的形态都是由一段可解释的“基因代码”实时生成的。
- 对我项目的启发：你的艺术技术背景在这里是杀手锏。不要只让 LLM 生成图片，要让它生成**生成图片的逻辑**。
- 今天是否值得深入研究：**是**
- 建议行动：做demo（尝试用 LLM 驱动 LOGO 或 p5.js 进行逻辑绘图）。

## 今日模式总结
1.  **空间化认知 (Spatial Cognition)**：将抽象的 LLM 记忆和逻辑，映射到物理空间（如 Minecraft 或 3D 坐标），利用人类对空间的直觉来增强 AI 的可解释性和检索效率。
2.  **演化式技能 (Evolving Skills)**：Agent 不再是静态的指令接收者，而是拥有动态生长的“技能代码库”，实现从执行者到“进化者”的转变。

## 今日可做小实验：Minecraft 记忆漫游者 (The Voxel Remembrancer)
这是一个将 MemPalace 简化并带入你熟悉的 Minecraft/世界生成背景的小实验。

- **项目名**：Voxel-Loci (体素记忆宫殿)
- **输入**：你的 10 条近期灵感笔记。
- **输出**：一个简单的 HTML/Three.js 场景（或 Minecraft 坐标列表），笔记被散布在不同的坐标点，坐标由 LLM 根据笔记的情感/主题计算得出。
- **技术栈**：Python, OpenAI API (Embedding), Three.js 或简单坐标映射脚本。
- **第一步怎么做**：
  1. 将 10 条笔记输入 `text-embedding-3-small` 获取向量。
  2. 使用 PCA（主成分分析）将 1536 维向量降至 3 维（即 X, Y, Z 坐标）。
  3. 在控制台打印出：`笔记 A 应放置在 [12, 64, -5] (森林区)`，并在一个简单的 3D 空间中点亮这些坐标。
  4. 尝试走近这些坐标，让 LLM 告诉你：“因为你现在处于这个空间位置，你最可能想到的灵感是...”

**评价**：这个实验能让你瞬间理解“空间隐喻”对 AI 记忆的改变，且非常符合你的跨学科背景！去试试吧！
