## 1. [MemPalace: LLM 记忆的空间隐喻架构]
- **来源：** arXiv cs.AI (2604.21284)
- **链接：** https://arxiv.org/abs/2604.21284
- **一句话解释：** 借鉴古希腊“记忆宫殿”法，将 LLM 的长效记忆映射到虚拟空间坐标中，实现极高精度的信息检索。
- **它为什么重要：** 解决了 RAG（检索增强生成）中“信息碎片化”和“语义迷失”的问题。它让数据不再是冷冰冰的向量，而是一个有结构的“地方”。
- **技术趋势标签：** #Spatial_Computing #RAG #HCI #Memory_Architecture
- **可复用模式：** **数据空间化 (Data Spatialization)**。将抽象的 Embedding 向量转化为具体的空间坐标或拓扑结构。
- **疯狂发明家灵感：**
  1. **一个小实验：** 用 Three.js 写一个 3D 仓库，把你的笔记存进去。当你问 AI 问题时，视角会瞬间切向那个存放笔记的“架子”。
  2. **一个中型项目：** **"The Infinite Library"**。一个自动生成的 Minecraft 地图，每一本书是一个知识点，书架的排列反映了知识的逻辑关联。
  3. **一个艺术/游戏/叙事方向：** 玩家在游戏中探索一个失忆 AI 的大脑。通过在空间中移动物体，玩家实际上是在重组 AI 的逻辑权重和记忆。
- **对我项目的启发：** 在做世界生成时，能否把“剧情线”设计成一种空间拓扑？
- **今天是否值得深入研究：** 是（其检索性能评估很有参考价值）。
- **建议行动：** **做 demo**（尝试将 Vector DB 的检索结果可视化为空间点云）。

## 2. [Co-Evolving LLM Decision and Skill Bank Agents]
- **来源：** arXiv cs.AI (2604.20987)
- **链接：** https://arxiv.org/abs/2604.20987
- **一句话解释：** 这种架构让 Agent 拥有一个“技能库”，在完成长距离任务（如玩复杂游戏）时，它会不断优化自己的技能代码，并学会如何串联这些技能。
- **它为什么重要：** 它是 **"Voyager" (Minecraft Agent)** 模式的进化版。它强调了“决策”与“技能代码”的协同进化。
- **技术趋势标签：** #Agent #Minecraft #Skill_Learning #Long_Horizon
- **可复用模式：** **代码即技能 (Code-as-Skill)**。Agent 不直接行动，而是写一段脚本存入库中，下次直接调用。
- **疯狂发明家灵感：**
  1. **一个小实验：** 写一个玩 P5.js 的 Agent，它每次画出一个好看的形状就将其封装成一个函数存入 `skill_bank.json`。
  2. **一个中型项目：** 一个自动化的“炼金术士”Agent，在虚拟实验室里组合不同的化学/魔法元素，通过不断尝试进化出复杂的合成配方脚本。
  3. **一个艺术/游戏/叙事方向：** **"The Self-Coding NPC"**。NPC 会根据玩家的反馈，在线重写自己的交互脚本，从而“进化”出适应玩家喜好的个性。
- **对我项目的启发：** 强化 Spec-driven development：让 Agent 先写出 API 定义（Spec），再填充实现，最后存入技能库。
- **今天是否值得深入研究：** 是。
- **建议行动：** **收藏**（这是目前最前沿的 Agent 架构思路）。

## 3. [WorkflowGen: 基于轨迹经验的自适应工作流生成]
- **来源：** arXiv cs.LG (2604.19756)
- **链接：** https://arxiv.org/abs/2604.19756
- **一句话解释：** 传统的 Agent 每次都要从头思考，而 WorkflowGen 会记录成功的“操作轨迹”，并将其固化为可复用的自动化工作流。
- **它为什么重要：** 解决了 Agent 运行时的“推理开销大”和“不稳定”问题。它把 Agent 从“思考者”转变为“熟练工”。
- **技术趋势标签：** #Automation #Workflow #Efficiency #Trajectory_Learning
- **可复用模式：** **轨迹固化 (Trajectory Distillation)**。从反复的尝试中提取出“黄金路径”。
- **疯狂发明家灵感：**
  1. **一个小实验：** 记录你手动处理 5 张图片的步骤（提示词、参数），让 LLM 提取出一个通用的 JSON 工作流。
  2. **一个中型项目：** 一个“设计助手”，它观察你在 Figma 里的操作，当你重复三次相似的排版后，它自动弹出一个按钮：“为你生成自动化排版工作流？”
  3. **一个艺术/游戏/叙事方向：** 游戏角色会学习玩家的“战斗风格”，并将其变成敌人的“标准战术”。
- **对我项目的启发：** 在 Drift 或自动化工具链中，引入一个“录制并抽象”的功能。
- **今天是否值得深入研究：** 否（看摘要理解思路即可）。
- **建议行动：** **试用**（尝试手动把一段 Agent 的 Trace 转化为一个固定函数）。

## 4. [Symbolic Grounding in Abstract Visual Reasoning (Bongard-LOGO)]
- **来源：** arXiv cs.AI (2604.21346)
- **链接：** https://arxiv.org/abs/2604.21346
- **一句话解释：** 探讨 VLM（视觉语言模型）是否真的懂抽象视觉？研究发现，如果让模型生成逻辑代码（LOGO 程序）来描述图像，其推理能力会大幅提升。
- **它为什么重要：** 它证明了 **“代码是思考抽象形状的最佳介质”**。这对于 Creative Coding 极其重要。
- **技术趋势标签：** #Creative_Coding #VLM #Symbolic_AI #Generative_Programs
- **可复用模式：** **视觉-代码对齐 (Visual-to-Code Grounding)**。不直接让模型描述图像，而是让它写出生成该图像的代码。
- **疯狂发明家灵感：**
  1. **一个小实验：** 给 GPT-4o 一张简单的几何抽象画，要求它写出一段能够完美复现该画作的 SVG 或 P5.js 代码。
  2. **一个中型项目：** **"The Visual Compiler"**。输入一段模糊的设计描述，Agent 通过不断生成代码并自检渲染结果，最终输出精确的设计系统。
  3. **一个艺术/游戏/叙事方向：** 一个基于 LOGO 语言的解谜游戏，玩家（或 Agent）必须通过编写几何逻辑程序来开启传送门。
- **对我项目的启发：** 这就是 Spec-driven development 在视觉领域的体现：图像是 Spec 的运行结果。
- **今天是否值得深入研究：** 是（对理解 VLM 的瓶颈非常有帮助）。
- **建议行动：** **做 demo**（视觉逆向工程：Image -> Code -> Image）。

## 5. [Do MLLMs Understand Pointing? (第一视角指向理解)]
- **来源：** arXiv cs.HC (2604.21461)
- **链接：** https://arxiv.org/abs/2604.21461
- **一句话解释：** 研究 MLLM 是否能理解智能眼镜视角下的“指指点点”。结论是：目前的模型很烂，它们往往在猜，而不是真的理解空间指向。
- **它为什么重要：** 这是 HCI 的圣杯：真正的空间交互。如果你想做 AR 或智能硬件 Agent，这是必须攻克的坎。
- **技术趋势标签：** #HCI #MLLM #Egocentric_Vision #Spatial_Reasoning
- **可复用模式：** **指代消解 (Referential Grounding)**。将手势、视线与环境中的物体实例建立关联。
- **疯狂发明家灵感：**
  1. **一个小实验：** 拍一张你桌子的照片，用手指着某个东西，看 Claude 3.5 能不能精准说出你指的东西的坐标。
  2. **一个中型项目：** 一个“空间指挥官”Agent。你在摄像头前指着屏幕上的代码块说：“重构这一段”，它能准确识别。
  3. **一个艺术/游戏/叙事方向：** **"Ghost in the Room"**。一个投影艺术装置，它能理解观众的指向动作，并根据观众指的位置生成特定的视觉反馈。
- **对我项目的启发：** 考虑在 Agent 工作流中引入“视觉反馈环”，而不仅仅是文本。
- **今天是否值得深入研究：** 否。
- **建议行动：** **忽略**（等模型能力提升后再回来看）。

## 今日模式总结
1. **空间化隐喻 (Spatial Metaphor)**：不再把信息看作列表，而是看作可以探索的 3D 场所（MemPalace）。
2. **代码即技能库 (Code-as-Skillset)**：Agent 的进化不再是权重的改变，而是其私有脚本库的扩张（Co-evolving Agents）。
3. **视觉即程序 (Vision-as-Program)**：理解抽象视觉的最好方式是将其转化为可执行的生成代码（Bongard-LOGO）。

## 今日可做小实验：The Desk Palace (桌面记忆宫殿)
**2 小时挑战：** 验证“空间记忆”是否能提升你对本地项目文件的检索感。

- **项目名：** DeskPalace-RAG
- **输入：** 你最近的一个项目文件夹（包含 .ts, .md, .json 文件）。
- **输出：** 一个 2D Canvas 网页，文件按照“语义相似度”分布在不同房间，点击房间可与该区域的文件内容对话。
- **技术栈：** Python (FastAPI) + Scikit-learn (PCA 降维) + Three.js / Canvas + OpenAI API。
- **第一步怎么做：**
  1. 用 `Sentence-Transformers` 提取文件夹内所有文件的 Embedding。
  2. 使用 `PCA` 或 `t-SNE` 将维度降到 2D 坐标 (x, y)。
  3. 在网页上把这些坐标画成点，点击点时，将该文件内容作为 Context 传给 LLM。
  4. **疯狂点：** 尝试手动把几个点挪动位置，告诉 LLM：“我把这两个概念放在一起了，以后它们就是相关的。”
