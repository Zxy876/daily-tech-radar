## 1. [MemPalace: LLM 记忆的空间隐喻架构]
- **来源：** arXiv cs.AI (2604.21284)
- **链接：** https://arxiv.org/abs/2604.21284
- **一句话解释：** 借用古老的“记忆宫殿”法，将 LLM 的长期记忆映射到虚拟空间坐标中，实现极高精度的检索。
- **它为什么重要：** 它把枯燥的 Vector DB 变成了**可感知的空间**。对于设计师来说，这意味着“数据可视化”直接变成了“数据导航”。
- **技术趋势标签：** #Spatial_Computing #LLM_Memory #HCI
- **可复用模式：** **空间索引化语义**（将 Embedding 向量映射到 3D 场景坐标）。
- **疯狂发明家灵感：**
  1. **一个小实验：** 用 Three.js 跑一个简单的房间，你丢给 AI 的每段对话都变成房间里的一个家具，位置由语义决定。
  2. **一个中型项目：** 一个 Minecraft 插件，玩家在世界里建造的建筑其实是 LLM 的知识库节点，拆除建筑等于删除记忆。
  3. **一个艺术/游戏/叙事方向：** “记忆考古”游戏。玩家进入一个 AI 的大脑（废墟都市），通过触摸不同的物件来提取 AI 遗忘的剧情碎片。
- **对我项目的启发：** 思考 Drift 或 Spec-driven 开发时，能否将“需求文档”变成一个**物理地图**？
- **今天是否值得深入研究：** **是**（这种空间感非常适合你的跨学科背景）。
- **建议行动：** 做 demo。

## 2. [Co-Evolving LLM Skill Bank: 长程任务的技能进化]
- **来源：** arXiv cs.AI (2604.20987)
- **链接：** https://arxiv.org/abs/2604.20987
- **一句话解释：** 让 Agent 在游戏环境中不仅是做决策，还能自己总结、优化并沉淀一套“技能库”（Skill Bank），像人类玩游戏一样越玩越溜。
- **它为什么重要：** 解决了 Agent “记不住教训”的问题。它让“经验”变成了可复用的“代码块”。
- **技术趋势标签：** #Agent_Workflow #Game_Systems #Self_Evolving
- **可复用模式：** **经验->代码化技能->库管理**。
- **疯狂发明家灵感：**
  1. **一个小实验：** 写一个 Python 脚本，让 LLM 玩一个简单的网格游戏，每成功一次就让它写一段函数存进 JSON，下次直接调用。
  2. **一个中型项目：** 一个“永不停止”的艺术生成 Agent，它在生成过程中不断总结自己的“构图技巧”并形成库。
  3. **一个艺术/游戏/叙事方向：** 一个会进化的 NPC 族群，它们通过相互交换“技能代码块”来学习如何在复杂的游戏世界中生存。
- **对我项目的启发：** 在做自动化系统时，不要只写 Prompt，要设计一个能让 Agent **自我迭代函数库**的机制。
- **今天是否值得深入研究：** **是**（非常契合 Minecraft/世界生成背景）。
- **建议行动：** 试用。

## 3. [WorkflowGen: 基于轨迹经验的自适应工作流生成]
- **来源：** arXiv cs.LG (2604.19756)
- **链接：** https://arxiv.org/abs/2604.19756
- **一句话解释：** 不再手动写复杂的 Agent 工作流（如 LangGraph），而是让系统观察人类的操作轨迹，自动生成并优化工作流。
- **它为什么重要：** **Spec-driven development 的终极形态**。你只需要展示“我想怎么做”，AI 帮你写出“自动化逻辑”。
- **技术趋势标签：** #Automation #Spec_driven #Workflow_Orchestration
- **可复用模式：** **轨迹捕获 -> 逻辑抽象 -> 流程固化**。
- **疯狂发明家灵感：**
  1. **一个小实验：** 录制一段你在浏览器里找资料的步骤（JSON 格式），让 LLM 把这段步骤转化成一个可执行的 Python 爬虫工作流。
  2. **一个中型项目：** 一个“影子设计师”工具，它学习你在 Figma 里的操作习惯，然后自动生成一套符合你风格的 UI 批量处理工作流。
  3. **一个艺术/游戏/叙事方向：** 交互式叙事生成器，根据玩家的前 10 分钟游戏行为，自动生成后续 1 小时的任务流逻辑。
- **对我项目的启发：** 减少手动配置 Workflow 的痛苦，尝试“以身示范”驱动生成。
- **今天是否值得深入研究：** 是。
- **建议行动：** 收藏。

## 4. [Do MLLMs Understand Pointing? 第一人称视角的指代推理]
- **来源：** arXiv cs.HC (2604.21461)
- **链接：** https://arxiv.org/abs/2604.21461
- **一句话解释：** 研究多模态模型是否能理解“指点”动作（如：指着桌上的杯子说“拿那个”），并提出了增强这种空间语义理解的方法。
- **它为什么重要：** 这是 HCI（人机交互）的痛点。如果 AI 懂“指哪打哪”，它就能真正进入物理世界或 VR 空间。
- **技术趋势标签：** #HCI #Multimodal #Spatial_Semantics
- **可复用模式：** **视觉坐标与语义标签的强耦合**。
- **疯狂发明家灵感：**
  1. **一个小实验：** 用摄像头拍一张你指着书架的照片，测试 GPT-4o 能否准确给出你手指末端对应的书名坐标。
  2. **一个中型项目：** 一个基于视觉的“魔法棒”工具，在房间里指着任何电器，LLM 会根据你的手势和语音自动调用该电器的 API。
  3. **一个艺术/游戏/叙事方向：** 纯手势驱动的生成艺术装置，观众的手势不仅是坐标，还是 LLM 生成画面的“笔触指令”。
- **对我项目的启发：** 在设计 Creative Tools 时，考虑如何利用“非文字”的输入（如指向、手势）来驱动 Agent。
- **今天是否值得深入研究：** 否（除非你现在要做硬件/VR 结合）。
- **建议行动：** 忽略（了解概念即可）。

## 5. [Symbolic Grounding: 抽象视觉推理中的符号瓶颈]
- **来源：** arXiv cs.AI (2604.21346)
- **链接：** https://arxiv.org/abs/2604.21346
- **一句话解释：** 证明了 VLM（视觉模型）在处理抽象图形逻辑时，如果能先生成“程序脚本”再推理，效果远好于直接看图说话。
- **它为什么重要：** 这验证了 **Creative Coding + LLM** 的优越性。用代码去描述视觉，比用像素去描述视觉更精准。
- **技术趋势标签：** #Creative_Coding #Generative_Systems #Reasoning
- **可复用模式：** **视觉 -> 符号/代码描述 -> 逻辑推理**。
- **疯狂发明家灵感：**
  1. **一个小实验：** 给 AI 一张复杂的几何艺术图，让它先写出生成这张图的 p5.js 代码，再问它图中圆形的个数。
  2. **一个中型项目：** 一个“草图转复杂 3D 模型”的编译器，AI 把草图理解为 OpenSCAD 代码，通过调整代码参数来实现精准变形。
  3. **一个艺术/游戏/叙事方向：** 基于 L-System 的生成艺术，LLM 作为“规则编写者”，通过修改符号规则来演化出一整个虚拟森林。
- **对我项目的启发：** 在处理生成式系统时，坚持“**代码作为中间层**”的思路，不要让 AI 直接出图。
- **今天是否值得深入研究：** **是**（对你的 Creative Coding 背景非常有益）。
- **建议行动：** 做 demo。

## 今日模式总结
1.  **空间化记忆 (Spatialization of Data)：** 记忆不再是列表，而是地图。这为 HCI 和游戏世界生成提供了天然的架构。
2.  **技能代码化 (Skills-as-Code)：** 成功的经验必须沉淀为可调用的函数库，Agent 才能实现长程任务的进化。
3.  **符号中介推理 (Symbolic Mediation)：** 处理视觉或复杂逻辑时，先转译为“代码/符号”是目前提升 LLM 准确性的最佳路径。

## 今日可做小实验：【LLM 记忆宫殿浏览器】
- **项目名：** Semantic Room Explorer
- **输入：** 你的 50 条最近的灵感笔记（文本）。
- **输出：** 一个简单的 HTML 页面，笔记根据语义被放置在 2D 画布的不同位置，相似的笔记靠在一起，形成“灵感群岛”。
- **技术栈：** OpenAI API (Embeddings), Python (Umap-learn 用于降维), HTML/Canvas (用于显示)。
- **第一步怎么做：**
  1. 调用 OpenAI Embedding 接口把你 50 条笔记转成向量。
  2. 使用 `umap-learn` 库将高维向量降维到 2D 坐标 (x, y)。
  3. 用 Python 生成一个简单的 HTML，把每条笔记写在一个 `<div>` 里，`style="left: x; top: y"`。
  4. 打开浏览器，看看你的灵感在空间上是如何分布的。
