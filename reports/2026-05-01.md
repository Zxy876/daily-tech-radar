## 1. Operating-Layer Controls for Onchain Language-Model Agents
- **来源：** arXiv cs.AI
- **链接：** https://arxiv.org/abs/2604.26091
- **一句话解释：** 这是一个在 21 天内让 3505 个由真实资金驱动的 AI Agent 在链上进行 ETH 交易的实战研究，重点在于如何给 Agent 加上“物理安全锁”。
- **它为什么重要：** 它是目前最接近“疯狂发明”的实验。它不讨论模型能力，而是讨论“操作层控制”——当 Agent 真的能花你的钱时，我们如何通过结构化控制和自然语言策略来约束它。
- **技术趋势标签：** #Onchain-Agent #Autonomous-Finance #Safety-Control
- **可复用模式：** **“金库模式 (Vault Control)”**：将 Agent 的决策权与资金拨付权分离，Agent 负责提议，结构化规则负责拦截。
- **疯狂发明家灵感：**
  1. **一个小实验：** 写一个脚本，给 Agent 10 块钱，让它在闲鱼或淘宝上寻找并“砍价”购买一个特定艺术品，设置最大预算拦截。
  2. **一个中型项目：** **“自动艺术赞助人”**：一个根据社交媒体热度自动给独立艺术家打赏或购买 NFT 的 Agent，它有自己的审美偏好设定。
  3. **一个艺术/游戏/叙事方向：** 建立一个“赛博角斗场”，观众下注，AI Agent 操纵真实资产进行策略博弈，输掉的 Agent 真的会“破产”并注销账户。
- **对我项目的启发：** 在做自动化系统时，不要只写 Prompt，要写“断路器 (Circuit Breaker)”逻辑。
- **今天是否值得深入研究：** 是
- **建议行动：** **做demo**（尝试给 Agent 一个带限额的虚拟卡接口）。

## 2. AGEL-Comp: A Neuro-Symbolic Framework for Compositional Generalization
- **来源：** arXiv cs.AI
- **链接：** https://arxiv.org/abs/2604.26522
- **一句话解释：** 提出了一种神经符号架构，解决 LLM Agent 在复杂交互环境中“顾头不顾腚”的泛化失败问题。
- **它为什么重要：** 纯 LLM 经常在逻辑组合上翻车。这个框架通过“符号化”动作，让 Agent 能够像搭积木一样组合复杂任务。这对 Minecraft 等世界生成和复杂游戏系统至关重要。
- **技术趋势标签：** #Neuro-Symbolic #Interactive-Agents #Procedural-Logic
- **可复用模式：** **“动作原子化 (Atomic Actions)”**：不让 LLM 直接输出结果，而是输出可被程序解析和校验的“符号指令”。
- **疯狂发明家灵感：**
  1. **一个小实验：** 在 Minecraft 中用符号定义“砍树”、“造桌子”、“合成”，让 Agent 组合这些动作完成一个它从未见过的复杂建筑。
  2. **一个中型项目：** **“规格驱动的关卡设计师”**：输入一个抽象的游戏玩法逻辑（符号），Agent 自动生成对应的 3D 空间布局。
  3. **一个艺术/游戏/叙事方向：** 一个基于逻辑符号的“魔法系统”，玩家组合咒语符号，AI 负责解释这些符号的视觉和数值后果。
- **对我项目的启发：** 强化了 spec-driven development 的信心——用符号约束 LLM 是唯一让 Agent 变可靠的路径。
- **今天是否值得深入研究：** 是
- **建议行动：** **试用**（研究其符号映射逻辑）。

## 3. Tree-of-Text: A Tree-based Prompting Framework
- **来源：** arXiv cs.HC
- **链接：** https://arxiv.org/abs/2604.26501
- **一句话解释：** 将枯燥的体育数据表，通过“树状 Prompt”结构，转化成富有叙事感的比赛报告。
- **它为什么重要：** 解决“幻觉”的最好方式是给数据建立结构。这种“树状生成”非常适合将复杂的世界观数据（如 Minecraft 的地形、NPC 关系）转化为流畅的叙事文本。
- **技术趋势标签：** #Prompt-Engineering #Narrative-Generation #Table-to-Text
- **可复用模式：** **“分层叙事展开”**：根节点是核心事实，分支是细节补充，叶子节点是修辞润色。
- **疯狂发明家灵感：**
  1. **一个小实验：** 输入一个简单的 CSV 账单，用树状 Prompt 把它生成一段“守财奴的日记”。
  2. **一个中型项目：** **“世界生成器叙事插件”**：从 Minecraft 的种子数据导出地图特征，自动撰写该地区的“地方志”。
  3. **一个艺术/游戏/叙事方向：** **“实时演化历史书”**：游戏中的每一个随机事件都挂在叙事树上，自动生成一部不断加厚的、逻辑自洽的文明史。
- **对我项目的启发：** 对处理生成式系统的输出很有用，可以用“树”来控制生成的颗粒度。
- **今天是否值得深入研究：** 否（看摘要和结构图即可）。
- **建议行动：** **收藏**（作为叙事生成的工具库）。

## 4. Vibe Check: Personality and Alignment in LLM-Based Agents
- **来源：** arXiv cs.HC
- **链接：** https://arxiv.org/abs/2509.09870
- **一句话解释：** 研究了 Agent 的“性格 (Personality)”如何影响用户在完成任务时的感知。
- **它为什么重要：** 很多 Agent 只有功能没有“灵魂”。这篇论文证明了性格对齐（Vibe Check）能显著提升协作效率，这在 HCI 和艺术技术领域是核心命题。
- **技术趋势标签：** #HCI #Persona-Design #User-Perception
- **可复用模式：** **“性格参数化 (Parametric Personality)”**：将 Agent 的性格作为可调节的变量，而不仅仅是 Prompt 里的形容词。
- **疯狂发明家灵感：**
  1. **一个小实验：** 写两个 Agent，一个极其暴躁，一个极其谄媚，观察它们在协助你写代码时的不同干扰效应。
  2. **一个中型项目：** **“情绪自适应编辑器”**：根据你的打字速度和删减频率，Agent 自动切换“鼓励型”或“严厉校对型”人格。
  3. **一个艺术/游戏/叙事方向：** **“性格驱动的 UI”**：Agent 的性格会改变 UI 的配色、字体和交互动效（如：傲慢的 Agent 会故意延迟响应）。
- **对我项目的启发：** 在做创意工具时，Agent 的“人设”也是功能的一部分。
- **今天是否值得深入研究：** 是
- **建议行动：** **做demo**（尝试给你的工具加上性格开关）。

## 5. How Do AI Agents Spend Your Money? (Token Consumption Analysis)
- **来源：** arXiv cs.HC
- **链接：** https://arxiv.org/abs/2604.22750
- **一句话解释：** 分析并预测 Agent 在编程等复杂任务中是如何“挥霍” Token 的。
- **它为什么重要：** Agent 递归调用产生的 Token 爆炸是开发者最大的焦虑。这篇论文提供了一个“Token 预算”的视角，甚至让 Agent 预测自己的消费。
- **技术趋势标签：** #Token-Economy #Agent-Efficiency #Cost-Prediction
- **可复用模式：** **“自省式预算 (Self-Predicting Budget)”**：在执行前让 Agent 估算成本，如果超过阈值则自动简化策略。
- **疯狂发明家灵感：**
  1. **一个小实验：** 写一个 Wrapper，在 Agent 运行前问它：“你觉得这个任务值多少钱？”，如果它的估算和实际相差太远，就触发惩罚 Prompt。
  2. **一个中型项目：** **“贫穷限制想象力”编辑器**：一个强制低成本的编程 Agent，它被限制只能用最小的 Context 完成最复杂的重构任务。
  3. **一个艺术/游戏/叙事方向：** **“能源危机游戏”**：玩家的每一个 AI 指令都要消耗有限的“赛博能源”（真实 Token 费），强制玩家进行精简的 spec-driven 思考。
- **对我项目的启发：** Token 消耗不只是成本，它可以变成一种游戏机制或约束条件。
- **今天是否值得深入研究：** 是
- **建议行动：** **试用**（监控你现有项目的 Token 消耗模式）。

## 今日模式总结
1.  **符号化操作层 (Symbolic Operating Layer)：** 纯文本生成已过时，现在的趋势是在 LLM 和真实世界（金钱、动作、代码）之间加一层“符号验证层”。
2.  **叙事的数据结构化 (Structural Narrative)：** 用树状或图状结构强行约束 LLM 的生成流，防止逻辑断层。
3.  **性格即功能 (Personality as Feature)：** 性格不再是装饰，而是影响交互效率和用户留存的硬指标。

## 今日可做小实验：【吝啬鬼叙事 Agent】
- **项目名：** Scrooge-Narrator
- **输入：** 一组枯燥的结构化数据（例如你的 GitHub Commit 记录或今日消费账单）。
- **输出：** 两个性格截然不同的 Agent 针对这组数据的“吵架式”解读。
- **技术栈：** Python + OpenAI API (GPT-4o-mini 即可) + 简单的树状 Prompt 模板。
- **第一步怎么做：**
  1. 准备一份你今天的消费账单 JSON。
  2. 编写两个 System Prompt：一个是“极其抠门且毒舌的会计”，一个是“今朝有酒今朝醉的艺术家”。
  3. 使用“树状 Prompt”：第一层提取消费总额，第二层分析大头，第三层让两个 Agent 针对具体的一笔消费进行性格化的争论。
  4. 观察结构化数据如何通过性格碰撞产生有趣的叙事。
