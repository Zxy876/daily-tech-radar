## 1. Wuphf: 代理人维护的 Git 知识基座
- 来源：Hacker News
- 链接：https://github.com/nex-crm/wuphf
- 一句话解释：一个让 Agent 自动读写的 Wiki 层，使用 Markdown + Git 作为真相来源，配以 SQLite 索引。
- 它为什么重要：Karpathy 一直在推崇“LLM 原生知识基座”。它抛弃了复杂的向量数据库，回归到开发者最熟悉的 Git 工作流。
- 技术趋势标签：#AgentInfrastructure #GitOps #MarkdownFirst
- 可复用模式：**“Git即记忆”**。利用 Git 的版本管理来追踪 Agent 的思维演进。
- 疯狂发明家灵感：
  1. **一个小实验**：写一个脚本，让 Agent 每天阅读你的笔记，并在 Git 上自动提交“矛盾点分析”和“知识合并”。
  2. **一个中型项目**：一个“自我进化的设计文档”。你只写 Spec，Agent 在 Git 上通过不断运行测试来自动补全并修正文档。
  3. **一个艺术/游戏/叙事方向**：**“活着的编年史”**。一个 Minecraft 服务器的后台 Agent，它观察玩家行为并实时在 Wiki 上撰写带有主观色彩的“史书”，玩家可以去 Git 提交 PR 挑战历史。
- 对我项目的启发：在做 spec-driven development 时，不要只把 spec 当输入，要让 Agent 把实现过程写回 spec 的分支。
- 今天是否值得深入研究：是
- 建议行动：试用

## 2. Matt Pocock 的 .claude 技能库
- 来源：GitHub Trending
- 链接：https://github.com/mattpocock/skills
- 一句话解释：知名开发者 Matt Pocock 公开了他给 Claude Code 使用的个人技能目录。
- 它为什么重要：这预示着一种新形态的**“程序员数字资产”**。未来我们不只是分享代码，而是分享“如何教 AI 写我的代码”的 Skill 文件。
- 技术趋势标签：#HCI #ClaudeCode #PromptEngineering
- 可复用模式：**“技能注入模式”**。通过标准化的 Markdown 定义 Agent 的工具使用边界。
- 疯狂发明家灵感：
  1. **一个小实验**：为你最常用的 Creative Coding 库（如 p5.js）写一套“黄金 Skill”，看它是否能瞬间提升 Claude 的绘图成功率。
  2. **一个中型项目**：**“风格克隆器”**。收集你过去所有项目的代码风格，提炼成一个 .skill，让任何 LLM 都能瞬间“夺舍”你的编程习惯。
  3. **一个艺术/游戏/叙事方向**：**“角色性格包”**。为不同的 NPC 定义专属的 Skill（例如“说谎技能”、“炼金技能”），这些 Skill 实际上是特定的 Prompt 指令集。
- 对我项目的启发：思考如何将你的“艺术审美”转化为 Agent 可执行的“审美规范 Skill”。
- 今天是否值得深入研究：是
- 建议行动：做demo（把你的一项绝活写成 Skill）

## 3. RLM: 递归语言模型推理库
- 来源：GitHub Trending
- 链接：https://github.com/alexzhang13/rlm
- 一句话解释：一个即插即用的推理库，支持递归式的语言模型调用。
- 它为什么重要：它在探索 LLM 如何通过“自我循环”和“沙盒反馈”来解决复杂逻辑，而不是简单的一次性输出。
- 技术趋势标签：#RecursiveInference #SelfCorrection #LLMChain
- 可复用模式：**“递归修正循环”**。
- 疯狂发明家灵感：
  1. **一个小实验**：让 LLM 写一段代码，然后递归地让它自己找出这段代码里的 5 个潜在 Bug，直到它找不出为止。
  2. **一个中型项目**：**“分形故事生成器”**。从一句话开始，递归地生成细节，每一层细节又成为下一层生成的背景，实现无限深度的叙事缩放。
  3. **一个艺术/游戏/叙事方向**：一个基于递归逻辑的**“哲学辩论 Agent”**。它不断反思自己的前一个观点，在控制台输出一场无止境的自我博弈。
- 对我项目的启发：在自动化工作流中引入“递归自省”环节，能大幅提高生成内容的质量。
- 今天是否值得深入研究：否（原理重要，库可以先观望）
- 建议行动：收藏

## 4. Martin Galway 的 C64 游戏音乐源码
- 来源：Hacker News
- 链接：https://github.com/MartinGalway/C64_music
- 一句话解释：80 年代传奇作曲家 Martin Galway 在 Commodore 64 上的音乐源代码公开。
- 它为什么重要：这是极简主义硬件下的极致艺术。对于关注生成式系统和游戏系统的你，这是研究“受限系统下的创造力”的绝佳案例。
- 技术趋势标签：#RetroTech #CreativeCoding #GameMusic
- 可复用模式：**“受限生成”**。在极小资源下构造复杂听感的逻辑。
- 疯狂发明家灵感：
  1. **一个小实验**：把这些 6502 汇编代码喂给 Claude，让它解释当时的作曲逻辑，并尝试用 JavaScript 重现一个 Web 版音色模拟器。
  2. **一个中型项目**：**“AI 复古作曲机”**。训练一个微型模型，专门学习这种 8-bit 源码的结构，生成现代 LLM 很难模拟出的纯正复古芯片音乐。
  3. **一个艺术/游戏/叙事方向**：一个 Minecraft 模组，根据玩家所在的生物群落，实时将环境数据转化为 C64 风格的代码并播放。
- 对我项目的启发：艺术技术不一定要追求“大”，回归到最底层、最受限的逻辑往往能产生最独特的风格。
- 今天是否值得深入研究：是
- 建议行动：试用（阅读源码，感受那种纯粹的逻辑美）

## 今日可做小实验：The Sentient Skill-Tree (有感知力的技能树)
这个实验结合了上述的“技能库”和“递归逻辑”，帮你快速上手 Agent 自动化。

- **项目名**：Skill-Crafter-Agent
- **输入**：一个简单的想法（例如：“我想做一个会画哥特式像素画的工具”）
- **输出**：一个自动生成的 `.claude/skills/pixel-art.md` 技能文件。
- **技术栈**：Claude API (或 Claude Code), Python/Node.js.
- **第一步怎么做**：
  1. 在本地创建一个 `.claude/skills` 文件夹。
  2. 编写一个系统提示词，要求 Agent：“分析我过去 2 小时的代码历史/对话历史，总结出我最常用的 3 个设计模式，并按照 Matt Pocock 的格式将其写成一个 Markdown 技能文件存入该目录。”
  3. 观察 Agent 是否能精准捕捉到你的“私人偏好”。
