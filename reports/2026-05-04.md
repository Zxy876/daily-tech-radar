## 1. ouroboros: 从“调教”转向“规格”的 Agent OS
- 来源：GitHub Trending
- 链接：https://github.com/Q00/ouroboros
- 一句话解释：一个提出“停止 Prompt，开始 Specify”口号的 Agent 操作系统。
- 它为什么重要：它代表了 **Spec-driven development** 的极致：不再用自然语言求 AI 办事，而是通过定义清晰的状态机和规格说明书让 Agent 运行。
- 技术趋势标签：#AgentOS #SpecDriven #SystemDesign
- 可复用模式：**“规格即执行”模式**。将复杂的任务拆解为可验证的 Spec 节点，而不是一段模糊的 Prompt。
- 疯狂发明家灵感：
  1. **一个小实验**：用 Markdown 写一个“自动点奶茶”的逻辑规格书，看它能否在不同 LLM 下保持逻辑一致。
  2. **一个中型项目**：开发一个 Minecraft 建筑 Agent，输入是“建筑蓝图规格 (JSON)”，Agent 负责在服务器里通过代码逐步实现。
  3. **一个艺术/游戏/叙事方向**：一个“规则驱动型”互动叙事系统。玩家不输入对话，而是修改世界的“物理法则”或“社会契约”规格，观察 Agent 居民的行为演变。
- 对我项目的启发：你的 Spec-driven 项目可以参考它的架构，将“意图”与“执行”完全解耦。
- 今天是否值得深入研究：是
- 建议行动：试用

## 2. ios-simulator-skill: 给 Claude 装上 iPhone 假肢
- 来源：GitHub Trending
- 链接：https://github.com/conorluddy/ios-simulator-skill
- 一句话解释：让 Claude Code 能够直接操控 iOS 模拟器，进行构建、运行和交互。
- 它为什么重要：打破了 LLM 只能写代码的僵局，它现在有了**视觉反馈和交互闭环**。这是 HCI 实验的绝佳工具。
- 技术趋势标签：#HCI #CreativeTool #MobileAgent
- 可复用模式：**“模拟器作为 Agent 身体”模式**。
- 疯狂发明家灵感：
  1. **一个小实验**：让 Claude 在模拟器里玩《2048》，看它如何通过截图反馈来做决策。
  2. **一个中型项目**：一个“UI 进化实验室”。设定一个审美目标，让 Agent 自动修改 SwiftUI 代码并在模拟器里运行，通过视觉识别打分，不断进化 UI。
  3. **一个艺术/游戏/叙事方向**：一个住在 iPhone 里的“虚拟生命”。它会根据你手机的通知、天气、电量，自动在模拟器里生成对应的视觉艺术或日记。
- 对我项目的启发：如果你在做 Creative Coding 或是自动化工具，这是一个现成的“手”，帮你触达移动端。

## 今日可做小实验：iOS 模拟器里的“数字生物”
这是一个 2 小时内能让你兴奋起来的跨学科实验。

- **项目名**：Sim-Life: UI Archeology (模拟生命：UI 考古)
- **输入**：Claude Code + `ios-simulator-skill` + 一个基础的 SwiftUI 模板。
- **输出**：一个在模拟器中根据“外界刺激”自动演化的 UI 界面。
- **技术栈**：Claude Code, Xcode Simulator, TypeScript/Swift.
- **第一步怎么做**：
    1.  安装 `ios-simulator-skill`。
    2.  启动 Claude Code，连接到你的 iOS 模拟器环境。
    3.  给 Claude 一个指令：“在模拟器里启动一个空白 App。现在，你是一个住在手机里的数字生物，你的‘心情’由我当前系统的电量决定（你可以读取它）。请你根据心情，实时修改这个 App 的 SwiftUI 代码，改变它的颜色、形状和动画。开始你的表演。”
    4.  观察 Claude 如何一边写代码，一边在模拟器里查看效果。
