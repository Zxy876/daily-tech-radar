## 1. 让图谱自己思考，而不是让 LLM 读图 (Don't Make the LLM Read the Graph)
- **来源**：arXiv cs.AI (2604.23057)
- **链接**：https://arxiv.org/abs/2604.23057
- **一句话解释**：在合作游戏《花火》(Hanabi) 中，研究者发现与其把复杂的逻辑关系写成 Prompt 塞给 LLM，不如构建一个“显式信念图谱”（Belief Graph），让图谱本身承载推理逻辑。
- **它为什么重要**：它解决了多智能体协作中“我知道你知道我知道”的这种套娃式推理难题。
- **技术趋势标签**：#Multi-Agent #Game-AI #TheoryOfMind #GraphReasoning
- **可复用模式**：**“外部化心智模型”**。不要试图用自然语言描述复杂的逻辑状态，而是用一个独立的数据结构（图）来维护 Agent 对世界的看法。
- **疯狂发明家灵感**：
  1. **一个小实验**：做一个“猜心机器人”，两人玩猜谜，AI 通过一个实时更新的图谱来推测对方掌握的信息。
  2. **一个中型项目**：Minecraft 里的“传闻系统”。NPC 之间不直接共享数据库，而是各自拥有一个“信念图谱”，通过对话交换图谱片段，产生误解、谎言和阴谋。
  3. **一个艺术/游戏/叙事方向**：一个互动装置，观众的动作会改变一个复杂的动态图谱，AI 试图通过解释这个图谱来讲述一个关于“误会”的故事。
- **对我项目的启发**：在做 Minecraft 世界生成或 Agent 协作时，给每个 Agent 挂载一个动态更新的 `knowledge_graph.json`，比写长 Prompt 更有用。
- **今天是否值得深入研究**：是
- **建议行动**：做 demo

## 2. AI 小说创作助手：从灵感到整本生产 (AI-Novel-Writing-Assistant)
- **来源**：GitHub Trending
- **链接**：https://github.com/ExplosiveCoderflome/AI-Novel-Writing-Assistant
- **一句话解释**：一个开源的 AI Native 小说创作系统，集成了世界观建模、RAG、写法引擎和 Agent 编排。
- **它为什么重要**：它把“写小说”这个感性行为拆解成了极其严密的“Spec-driven”工程工作流。
- **技术趋势标签**：#Workflow #AgenticCreativeTool #RAG #NarrativeSystems
- **可复用模式**：**“分层创作架构”**（灵感 -> 设定 -> 大纲 -> 章节 -> 润色）。
- **疯狂发明家灵感**：
  1. **一个小实验**：写一个 Python 脚本，输入一个“世界观设定”，自动生成该世界的“物理定律”和“禁忌清单”。
  2. **一个中型项目**：一个“全自动桌游生成器”。基于该项目的写法引擎，不仅生成背景故事，还自动生成对应的数值数值和卡牌效果。
  3. **一个艺术/游戏/叙事方向**：一个“永恒小说”直播间。Agent 不断根据弹幕反馈修改世界观，并实时生成和朗读小说，观众可以投票决定某个角色的命运。
- **对我项目的启发**：关注它的“世界观引擎”是如何保持长文本一致性的，这直接可以用在游戏叙事生成中。
- **今天是否值得深入研究**：是
- **建议行动**：试用

## 3. PhySE：基于 AR+LLM 的心理社会工程攻击
- **来源**：arXiv cs.AI (2604.23148)
- **链接**：https://arxiv.org/abs/2604.23148
- **一句话解释**：利用 AR 眼镜捕捉视觉和语音，LLM 实时分析目标的心理状态，并指导佩戴者如何进行“社会工程学”操纵。
- **它为什么重要**：虽然它是关于攻击的研究，但它展示了 **HCI + AI 实时反馈** 在社交场景中的恐怖潜力。
- **技术趋势标签**：#HCI #AR #SocialEngineering #RealTimeAI
- **可复用模式**：**“社交增强现实（Social Augmentation）”**。将 LLM 作为人类感官的“心理分析插件”。
- **疯狂发明家灵感**：
  1. **一个小实验**：用手机摄像头对准电视里的访谈节目，让 LLM 实时分析嘉宾的“潜台词”并弹窗提示。
  2. **一个中型项目**：一个“社交恐惧症辅助眼镜”。AR 识别对方表情，AI 在镜片上提示：“他现在有点无聊，建议换个关于猫的话题”。
  3. **一个艺术/游戏/叙事方向**：一个沉浸式戏剧。演员戴着 AR 眼镜，观众通过后台修改 AI 的指令来控制演员的表演情绪。
- **对我项目的启发**：思考“穿戴式 Agent”如何改变人类的即时决策，这比单纯的网页 Agent 更有趣。
- **今天是否值得深入研究**：是
- **建议行动**：做 demo

## 4. 1-Bit 信号驱动的智能体安全规范发现 (EPO-Safe)
- **来源**：arXiv cs.AI (2604.23210)
- **链接**：https://arxiv.org/abs/2604.23210
- **一句话解释**：Agent 仅通过极其稀疏的“1-Bit 危险信号”（对/错，红灯/绿灯），就能通过迭代反射进化出复杂的行为规范。
- **它为什么重要**：证明了“极简反馈”也能驱动“复杂规范”的生成。这对自动化系统和 Spec-driven 开发极具启发。
- **技术趋势标签**：#ReinforcementLearning #Reflection #SpecEvolution #Automation
- **可复用模式**：**“反馈驱动的 Spec 演化”**。不需要写复杂的 Prompt，只需要给 AI 报错，让它自己总结“为什么错”。
- **疯狂发明家灵感**：
  1. **一个小实验**：写一个生成艺术脚本，你只按“空格”表示喜欢，让 AI 自动总结出你的“审美 Spec”并生成更多。
  2. **一个中型项目**：一个“自我修复的代码机器人”。它尝试运行代码，只要报错（1-bit 失败信号），它就反思并修改自己的行为准则，直到通过。
  3. **一个艺术/游戏/叙事方向**：一个“盲盒生物”。玩家无法直接控制它，只能给它“正/负”反馈，生物会逐渐进化出独特的人格和行为逻辑。
- **对我项目的启发**：在做自动化工作流时，不要试图预设所有规则，建立一个错误反馈回路，让 Agent 自己写 Spec。
- **今天是否值得深入研究**：是
- **建议行动**：收藏

## 5. StoryTR：带“心智理论”的叙事视频检索
- **来源**：arXiv cs.AI (2604.23198)
- **链接**：https://arxiv.org/abs/2604.23198
- **一句话解释**：不仅识别视频里发生了什么（What），还能推理为什么发生（Why），即赋予 AI 推断角色意图、心理状态和叙事因果的能力。
- **它为什么重要**：这是从“视觉识别”到“叙事理解”的跨越。
- **技术趋势标签**：#TheoryOfMind #NarrativeReasoning #VideoAI
- **可复用模式**：**“意图推断（Intent Inference）”**。在生成系统中，不仅生成动作，还要生成动作背后的“动机”。
- **疯狂发明家灵感**：
  1. **一个小实验**：把一段 30 秒的电影片段喂给 LLM，让它写出每个角色的“内心独白”。
  2. **一个中型项目**：一个“游戏录像自动解说员”。它不解说“他开了一枪”，而是解说“他因为害怕被包抄，所以选择火力压制”。
  3. **一个艺术/游戏/叙事方向**：一个“基于心智的 Minecraft NPC”。NPC 会根据玩家的动作推测玩家的意图（是想送礼物还是想抢东西），并据此产生情绪反应。
- **对我项目的启发**：在设计游戏系统或叙事系统时，增加一个“意图层（Intent Layer）”，让系统更像人。
- **今天是否值得深入研究**：是
- **建议行动**：收藏

## 今日模式总结
1. **显式心智模型 (Explicit Mind Modeling)**：不再迷信 LLM 的黑盒推理，而是通过图谱、意图层等显式结构来增强 Agent 的理解力。
2. **极简反馈闭环 (Minimalist Feedback Loops)**：利用 1-bit 或稀疏信号驱动复杂行为的演化，这在自动化和生成艺术中极其高效。
3. **社交/叙事增强 (Social/Narrative Augmentation)**：AI 正在从“替代人类工作”转向“增强人类感知和表达”。

## 今日可做小实验
- **项目名**：**1-Bit 审美进化器 (1-Bit Aesthetic Evolver)**
- **输入**：用户对 AI 生成图片的简单“喜欢/不喜欢”点击。
- **输出**：一个不断进化的、用自然语言描述的“审美准则文本（Aesthetic Spec）”。
- **技术栈**：Python + OpenAI API (GPT-4o) + 任意生成艺术库 (如 p5.js 或简单的 SVG 生成)。
- **第一步怎么做**：
  1. 编写一个 Prompt，让 LLM 根据当前的 `Aesthetic_Spec`（初始为空）生成一段 p5.js 代码。
  2. 展示生成的图像，你点击“👍”或“👎”。
  3. 将反馈传给 LLM：如果是“👍”，让它总结当前代码中成功的元素；如果是“👎”，让它反思失败的原因。
  4. 更新 `Aesthetic_Spec`，循环往复。看看 10 次点击后，它是否能总结出你喜欢的风格。
