## 1. YourMemory: 具有“生物遗忘曲线”的 AI 记忆
- 来源：Hacker News
- 链接：[https://github.com/sachitrafa/YourMemory](https://github.com/sachitrafa/YourMemory)
- 一句话解释：利用艾宾浩斯遗忘曲线模型，让 AI 像生物一样“忘记”无关紧要的细节，只保留重要记忆。
- 它为什么重要：当前的 RAG（检索增强生成）像个永不清理的堆填区，越久越乱。引入“时间衰减”和“复习强化”让 AI 的认知更像人类，能极大降低上下文噪音。
- 技术趋势标签：#BiomimeticAI #MemoryManagement #HCI
- 可复用模式：**基于权重的动态遗忘机制**。
- 疯狂发明家灵感：
  1. **一个小实验**：写一个脚本，根据你与 AI 聊天的频率，自动删除那些一个月没被提及的对话片段，看它的回复是否变得更“精干”。
  2. **一个中型项目**：一个“数字痴呆”日记本。AI 会记录你的生活，但如果你不经常翻看某段往事，那段记忆的文字会逐渐模糊（甚至被替换成随机生成的虚假记忆）。
  3. **一个艺术/游戏/叙事方向**：一个会“老去”的 NPC。随着游戏时间的推移，NPC 会忘记任务细节，甚至忘记玩家的名字，除非玩家通过特定的行为“唤醒”其记忆。
- 对我项目的启发：在 Minecraft 代理中，让 Agent 忘记 1000 步以前的琐碎坐标，只记住“家”和“矿坑”的位置。
- 今天是否值得深入研究：是
- 建议行动：做demo（尝试将其逻辑接入你现有的 Agent 框架）。

## 2. wuphf: 代理维护的 Karpathy 式 Wiki
- 来源：Hacker News Best
- 链接：[https://github.com/nex-crm/wuphf](https://github.com/nex-crm/wuphf)
- 一句话解释：一个以 Markdown 和 Git 为底层的知识库，由 AI 代理自动维护、索引和读写。
- 它为什么重要：它实践了“LLM-native knowledge substrate”的概念。知识不再是死的文件，而是 Agent 可以自主呼吸和进化的土壤。
- 技术趋势标签：#SpecDriven #KnowledgeGraph #GitOps
- 可复用模式：**Markdown-as-Truth (将文档作为 Agent 的唯一真理来源)**。
- 疯狂发明家灵感：
  1. **一个小实验**：建立一个本地文件夹，让 Agent 每天根据你的浏览记录更新一个 `daily_spec.md`，并自动 Git Commit。
  2. **一个中型项目**：一个“自我进化的说明书”。当你修改代码时，Agent 自动检测差异并重写对应的 Markdown 文档，确保文档永远不落后于代码。
  3. **一个艺术/游戏/叙事方向**：一个“活着的设定集”。一群 Agent 共同生活在一个 Markdown 文件夹里，通过相互修改对方的 .md 文件来模拟社会关系的变迁。
- 对我项目的启发：这就是 spec-driven development 的终极形态：代码即文档，文档即 Agent 的大脑。
- 今天是否值得深入研究：是
- 建议行动：试用（克隆下来看看它如何处理 Git 冲突）。

## 3. GitNexus: 浏览器内的代码知识图谱
- 来源：GitHub Trending
- 链接：[https://github.com/abhigyanpatwari/GitNexus](https://github.com/abhigyanpatwari/GitNexus)
- 一句话解释：无需服务器，直接在浏览器里把 GitHub 仓库变成一个交互式的、带 Graph RAG Agent 的知识图谱。
- 它为什么重要：它把“理解代码”变成了一种视觉探索。对于跨学科学生来说，这是将复杂的 CS 逻辑“空间化”的最佳工具。
- 技术趋势标签：#DataViz #GraphRAG #CreativeCoding
- 可复用模式：**客户端代码分析 + 视觉映射**。
- 疯狂发明家灵感：
  1. **一个小实验**：把自己的一个混乱项目丢进去，看看 Agent 总结出的逻辑核心是否和你预想的一样。
  2. **一个中型项目**：一个“代码星系”生成器。根据代码的调用频率和复杂度，生成一个 3D 的可交互星系，文件是恒星，调用是引力线。
  3. **一个艺术/游戏/叙事方向**：把代码库变成 Minecraft 里的城市。函数是建筑，依赖关系是道路，Agent 在其中巡逻维护。
- 对我项目的启发：利用 Graph RAG 来增强你对复杂系统（如游戏引擎）的理解。
- 今天是否值得深入研究：是
- 建议行动：收藏（作为视觉化工具库储备）。

## 4. cua: 计算机使用代理（Computer-Use Agents）基础设施
- 来源：GitHub Trending
- 链接：[https://github.com/trycua/cua](https://github.com/trycua/cua)
- 一句话解释：为那些能直接控制操作系统（macOS/Linux/Windows）的 Agent 提供沙盒、SDK 和评估基准。
- 它为什么重要：这是 HCI 的下一个前沿。Agent 不再局限于对话框，而是直接操作 GUI（点图标、拖文件）。
- 技术趋势标签：#HCI #Automation #AgenticUI
- 可复用模式：**视觉反馈回路（看屏幕 -> 思考 -> 模拟点击）**。
- 疯狂发明家灵感：
  1. **一个小实验**：写一个简单的 Agent，让它在你的电脑上自动打开 Figma，画一个圆，然后保存。
  2. **一个中型项目**：一个“艺术助手”。当你画画时，它在后台帮你整理图层、自动根据你的草图搜索素材并拖进画布。
  3. **一个艺术/游戏/叙事方向**：一个“幽灵用户”。在你的电脑闲置时，Agent 模拟人类操作进行“创作”或“整理”，你回来时会发现桌面变了样。
- 对我项目的启发：思考“自动化系统”如何从 API 调用转向视觉操作。
- 今天是否值得深入研究：否（基础设施较重，除非你现在就要做 GUI 自动化）。
- 建议行动：忽略（先关注更轻量的逻辑实现）。

## 5. rlm: 递归语言模型推理库
- 来源：GitHub Trending
- 链接：[https://github.com/alexzhang13/rlm](https://github.com/alexzhang13/rlm)
- 一句话解释：让语言模型通过“递归”的方式自我推理和修正，支持各种沙盒环境。
- 它为什么重要：递归是解决复杂生成问题的利器。它能让 LLM 不断深挖一个概念，直到达到某种“终止条件”。
- 技术趋势标签：#RecursiveAI #GenerativeSystems
- 可复用模式：**自我提纯（Refinement Loop）**。
- 疯狂发明家灵感：
  1. **一个小实验**：给 AI 一个极其模糊的艺术命题，让它递归 5 次，每次都要求“比上一次更疯狂、更具体”。
  2. **一个中型项目**：一个“无限世界生成器”。从一个词开始（如“森林”），递归生成地形、植物、动物、文化，直到形成一个完整的 Minecraft 世界规格说明。
  3. **一个艺术/游戏/叙事方向**：一个“梦中梦”叙事 Agent。每一层递归代表一个更深层的梦境，每一层都有不同的逻辑规则。
- 对我项目的启发：在生成复杂系统（如游戏关卡）时，利用递归来确保逻辑的严密性。
- 今天是否值得深入研究：是
- 建议行动：试用（写一段递归 Prompt 看看效果）。

## 今日可做小实验：【遗忘日记】
- **项目名**：Ebbinghaus Daily (艾宾浩斯日记 Agent)
- **输入**：你今天的一段随笔或一段代码思路。
- **输出**：一个会自动随时间“褪色”的网页或 Markdown 文件。
- **技术栈**：Python + OpenAI API + Markdown。
- **第一步怎么做**：
  1. 创建一个 JSON，存储你的文本和 `timestamp`。
  2. 写一个简单的公式：`visibility = 1 / (1 + log(now - timestamp))`。
  3. 调用 LLM，要求它：“根据 visibility 的数值（0-1），如果数值较低，请用模糊的词汇替换原句中的关键词；如果数值极低，请完全删除该段落。”
  4. 运行脚本，看看你的记忆是如何在 2 小时模拟中逐渐“风化”的。
